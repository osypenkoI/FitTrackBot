# FitTrackBot

FitTrackBot – це Telegram-бот для моніторингу та аналізу фізичної активності користувача. Бот дозволяє вести облік тренувань і харчування, переглядати статистику, формувати звіти, налаштовувати цілі та нагадування, а також виконувати прогнозування показників активності на основі методів обробки часових рядів.

## Основні можливості

– реєстрація користувача та налаштування профілю;
– введення даних про фізичну активність;
– облік харчування та калорій;
– перегляд історії активності та харчування;
– побудова аналітики та графіків;
– прогнозування активності на основі часових рядів;
– виявлення аномальних відхилень у даних;
– формування PDF та Excel-звітів;
– встановлення персональних цілей;
– налаштування нагадувань;
– соціальні функції, челенджі та лідерборди;
– службові endpoint-и для моніторингу через Zabbix.

## Вимоги до запуску

– Python 3.10 або вище;
– PostgreSQL 14 або вище;
– Telegram Bot Token, отриманий через BotFather;
– доступ до мережі Інтернет;
– встановлені залежності з файлу `requirements.txt`.

## Налаштування середовища

Перед запуском потрібно створити файл `.env` у кореневій папці проєкту.

Приклад вмісту `.env`:

```env
BOT_TOKEN=your_telegram_bot_token
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/fittrackbot
ADMIN_IDS=123456789
```

Файл `.env` містить конфіденційні дані та не повинен завантажуватися у GitHub.

## Встановлення залежностей

```bash
pip install -r requirements.txt
```

## Запуск бази даних PostgreSQL через Docker

Для PowerShell:

```powershell
docker run --name fittrackbot-postgres `
  -e POSTGRES_USER=postgres `
  -e POSTGRES_PASSWORD=postgres `
  -e POSTGRES_DB=fittrackbot `
  -p 5432:5432 `
  -d postgres:15
```

## Запуск Telegram-бота

```bash
python -m src.bot
```

Після запуску бот підключається до Telegram Bot API у режимі Long Polling та створює необхідні таблиці бази даних.

## Запуск службового сервера моніторингу

```bash
python -m uvicorn src.health_server:app --host 0.0.0.0 --port 8000
```

Доступні службові endpoint-и:

| Endpoint | Метод | Призначення |
|---|---|---|
| `/health` | GET | Перевірка доступності серверної частини |
| `/response-time` | GET | Отримання часу відповіді |
| `/memory` | GET | Отримання поточного використання оперативної пам’яті |
| `/cpu` | GET | Отримання поточного використання CPU |

## Навантажувальне тестування

Для імітації одночасної роботи користувачів використовується скрипт:

```bash
python load_test.py
```

Скрипт надсилає паралельні запити до службових endpoint-ів та дозволяє отримати телеметричні дані для подальшого аналізу в Zabbix.

## Тестування

Для запуску unit-тестів використовується команда:

```bash
pytest
```

Тести перевіряють роботу основних сервісів, валідаторів, аналітичного модуля та логіки обробки даних.

## Безпека

– токен Telegram-бота зберігається тільки у файлі `.env`;
– файл `.env` додано до `.gitignore`;
– конфіденційні параметри не публікуються у репозиторії;
– робота з базою даних виконується через SQLAlchemy;
– вхідні дані перевіряються через Pydantic.
